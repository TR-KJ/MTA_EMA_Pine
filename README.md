# MTAにセットアップ後、EMAブレイクでエントリーを検証
　ベースは、v8.1α系を使用

##トリガーパターン
- 案1：ゾーン接触後、終値がEMAを上抜け/下抜けしたらTrigger:
メリットは、見た目で確認しやすいこと。
デメリットは、EMA付近で細かいダマシが出やすいことです。
- 案2：ゾーン接触後、終値がEMAの上/下に確定したらTrigger:
- 案3：ゾーン接触後、終値がEMAを上抜け/下抜け、かつEMA傾きが有利方向
案1に傾きフィルターを足したものです。:
メリットは、案1より少し方向感が入ること。
デメリットは、トリガーが少し遅れる可能性があることです。
-案4：ゾーン接触後、短期EMAと長期EMAがクロスしたらTrigger:
メリットは、ノイズを少し減らしやすいこと。
デメリットは、1分足では遅れやすいことです。

これは「最初」よりも2番目か3番目の検証向きです。
-案5：EMAはTriggerではなくFilterとして使う

これは、将来Pivot Break Triggerや別トリガーと組み合わせる案です。:
メリットは、既存のB系トリガーと相性がいいこと。
デメリットは、今回は「EMAトリガー単体検証」なので、初回には向きません。
